# Welcome to First Build

**You're about to build your first real software project.** Not a guided demo. Not a fill-in-the-blank exercise. A real thing that you choose, build, and can show people when you're done.

## Who This Is For

You're comfortable with computers (you can install apps, use a browser, manage files) but you've never written code. Maybe you've been curious for a while. Maybe someone told you you should try it. Either way, you're here now, and that's the hardest part.

## What You'll Have When You're Done

- **A working development environment** with the same tools professional developers use every day
- **A finished project** you built yourself (a website, an app, a bot, or a tool)
- **Real skills** including Git, the terminal, at least one programming language, and good habits around security and documentation
- **A portfolio piece** that you can put on GitHub and share

## How This Works

This guide has two parts:

1. **The Setup Guide (this wiki):** You're reading it now. It walks you through installing and configuring everything you need, step by step. Budget about 1 to 2 hours. Go in order because each page builds on the last.

2. **Claude Code (your AI tutor):** Once your environment is set up, you'll open a terminal, type `claude`, and start building. Claude Code is an AI assistant that reads the instruction file in your project and becomes your personal tutor. It explains every concept before using it, quizzes you to make sure you understand, and adapts to your pace.

## What to Expect

- **You will make mistakes.** That's not a warning, it's a promise. Every working developer makes mistakes constantly. The difference between a beginner and an expert isn't that experts don't make mistakes; it's that they know how to find and fix them. You'll learn that too.
- **You don't need to understand everything right now.** Some things will click immediately. Others will make sense later, once you've used them a few times. That's normal.
- **Go at your own pace.** There's no clock. If a step takes you longer than expected, that's fine. If something doesn't make sense, ask Claude Code to explain it differently.
